Переписать ChannelCard.tsx, сохраняя общую структуру компонента. Конкретные изменения:

**1. Расширить ChannelCardProps:**
```typescript
interface ChannelCardProps {
  channel: ChannelData;
  className?: string;
  onActivate?: (stopFn: () => void) => void;   // single-active-player
  onDeactivate?: () => void;                     // при остановке стрима
}
```

**2. Заменить `startStream` на `fetchAndStartStream(signal: AbortSignal)`:**
- Сначала делает `fetch(`/api/tv/stream/${channel.id}`, { signal })` для получения stream URL
- При `!res.ok` или `signal.aborted` — silent return (thumbnail остаётся)
- Парсит ответ как `{ streamUrl?: string }`, если нет streamUrl — return
- Затем проверяет `.m3u8` и использует существующую логику:
  - Safari: нативный HLS через `video.src = streamUrl`
  - Другие: `hls.js` через dynamic import
- Изменить `startLevel: -1` на `startLevel: 0` (быстрый старт для preview)
- `setVideoReady(true)` ТОЛЬКО в обработчике события `playing` (не `loadedmetadata`)
- Все addEventListener с `{ signal }` для автоматической отмены

**3. Обновить `handleMouseEnter`:**
```typescript
const handleMouseEnter = useCallback(() => {
  setIsHovered(true);
  if (hoverTimerRef.current) clearTimeout(hoverTimerRef.current);
  hoverTimerRef.current = setTimeout(() => {
    abortRef.current?.abort();
    const ac = new AbortController();
    abortRef.current = ac;
    onActivate?.(() => {
      ac.abort();
      stopStream();
    });
    fetchAndStartStream(ac.signal);
  }, 200);
}, [fetchAndStartStream, stopStream, onActivate]);
```

**4. Обновить `handleMouseLeave`:**
- Добавить вызов `onDeactivate?.()` после `stopStream()`

**5. Crossfade — исправить duration (HOVER-05):**
- Заменить `duration-300` на `duration-[400ms]` в className video overlay
- Убрать inline `transitionDelay` — задержка не нужна, transition duration 0.4s сама по себе создаёт плавный переход

**6. LIVE badge — исправить условие (HLS-06):**
- Заменить `channel.streamUrl && isHovered` на `videoReady && isHovered`
- Бейдж показывается только когда видео РЕАЛЬНО играет

**7. Video overlay — всегда рендерить:**
- Убрать условие `{channel.streamUrl && (...)}` вокруг video overlay div
- Video element должен быть всегда в DOM (streamUrl теперь динамический, не проп)
- Условная видимость через opacity (уже реализовано)

**8. stopStream — сохранить как есть:**

**9. Cleanup useEffect — сохранить как есть:**
