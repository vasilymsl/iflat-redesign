<objective>
Реализовать CSS-архитектуру для 140%-расширения карточек каналов при hover.

Output: Карточки каналов на /tv корректно расширяются до 140% при hover, не обрезаются контейнером, анимируются плавно, не конфликтуют с header.

  <name>Task 1: CSS-архитектура для 140%-расширения карточек</name>
  <files>iflat-redesign/src/app/globals.css</files>
  <action>
Внести следующие изменения в globals.css. Ориентироваться на полный CSS-diff из 01-RESEARCH.md (секция "Code Examples > Полный CSS diff").

**1. `.tv-shelf__slider` (строка ~133) — добавить isolation:**
```css
.tv-shelf__slider {
  position: relative;
  isolation: isolate;  /* NEW: stacking context, z-index детей не конфликтуют с header z-50 */
}
```

**2. `.tv-shelf__scroll` (строка ~138) — убрать scroll-snap, добавить overflow-y:**
```css
.tv-shelf__scroll {
  display: flex;
  gap: 10px;
  overflow-x: auto;
  overflow-y: visible;       /* NEW: разрешает карточкам расти вертикально */
  /* scroll-snap-type: x mandatory — УБРАТЬ для десктопа */
  scroll-behavior: smooth;
  scrollbar-width: none;
  padding: 10px 0 2rem;
  margin: 0 3.5rem;
}
```

**3. Мобильный media query для `.tv-shelf__scroll` (строка ~150) — вернуть scroll-snap:**
Заменить существующий мобильный media query на:
```css
@media (max-width: 767px) {
  .tv-shelf__scroll {
    scroll-snap-type: x mandatory;  /* мобиль: snap полезен */
    overflow-y: auto;               /* мобиль: overflow-y auto (hover нет) */
    gap: 8px;
    padding: 8px 0 1.5rem;
    margin: 0 1rem;
  }
}
```

**4. `.channel-card` (строка ~206) — добавить z-index, убрать scroll-snap-align:**
```css
.channel-card {
  flex: 0 0 calc((100% - 10px * 5) / 6);
  min-width: calc((100% - 10px * 5) / 6);
  /* scroll-snap-align: start — УБРАТЬ на десктопе */
  position: relative;
  cursor: pointer;
  z-index: 1;               /* NEW: базовый z-index для работы hover z-index:20 */
}
```

**5. Мобильный media query для `.channel-card` (строка ~226) — вернуть snap-align:**
Добавить `scroll-snap-align: start;` в мобильный media query для `.channel-card` (max-width: 767px).

**6. `.channel-card-inner` (строка ~233) — убрать overflow:hidden, заменить transition:**
```css
.channel-card-inner {
  border-radius: 8px;
  position: relative;
  /* overflow: hidden — УБРАНО: обрезало бы расширение */
  width: 100%;              /* NEW: базовая ширина для анимации */
  transition: width 0.25s ease-out, box-shadow 0.25s ease-out, transform 0.25s ease-out;
  transition-delay: 0.2s;   /* NEW: 200ms hover intent delay (HOVER-02) */
}
```
ВАЖНО: `overflow: hidden` обязательно убрать. Но при этом `.channel-card__preview` (строка ~255) УЖЕ имеет `overflow: hidden` — его оставить, он нужен для aspect-ratio.

**7. Hover state (строка ~240) — заменить scale(1.05) на width:140%:**
Заменить весь блок `@media (hover: hover) and (min-width: 768px)` для channel-card:
```css
@media (hover: hover) and (min-width: 768px) {
  .channel-card:hover {
    z-index: 20;             /* NEW: поднять карточку над соседями */
  }

  .channel-card:hover .channel-card-inner {
    width: 140%;             /* NEW: расширение на 140% (HOVER-01) */
    transform: translateY(-6px);  /* NEW: lift-эффект */
    box-shadow: 0 16px 48px rgba(0, 0, 0, 0.7);
    transition-delay: 0s;   /* Анимация начинается сразу (delay уже отработал на default state) */
  }
}
```

**8. `.channel-card-inner--hovered` (строка ~250) — убрать или оставить пустым:**
Убрать `z-index: 5` — z-index теперь на outer `.channel-card:hover`. Класс можно оставить для будущего использования (Phase 3), но без стилей:
```css
.channel-card-inner--hovered {
  /* z-index управляется через .channel-card:hover */
}
```
Или убрать этот блок целиком — но тогда в ChannelCard.tsx класс будет применяться без эффекта (безвредно).

**Чего НЕ делать:**
- НЕ трогать `.channel-card__preview` (overflow:hidden там НУЖЕН для aspect-ratio)
- НЕ трогать `.content-card` (постеры "Новинки" — другой phase)
- НЕ добавлять `will-change: transform` на slider (создаст stacking context, конфликтует с isolation)
- НЕ использовать `scaleX(1.4)` — растягивает текст и логотипы (см. anti-patterns в RESEARCH.md)
