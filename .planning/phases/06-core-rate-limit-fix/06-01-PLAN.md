**3. Переписать `curlJson` (строки 260-286) с Promise-chain mutex:**
Использовать паттерн из RESEARCH.md (Паттерн 2 — упрощённый):

```typescript
async function curlJson(method: string, url: string, body?: unknown): Promise<unknown> {
  // 1. Синхронно захватить слот в очереди (ДО любого await!)
  const waitForPrev = requestQueue;
  let releaseSlot!: () => void;
  requestQueue = new Promise<void>(resolve => { releaseSlot = resolve; });

  // 2. Дождаться предыдущего
  await waitForPrev;

  // 3. Задержка rate-limit
  await sleep(DELAY_BETWEEN_REQUESTS_MS);

  // 4. Выполнить с retry
  for (let attempt = 0; attempt <= MAX_RETRIES; attempt++) {
    try {
      const result = curlJsonSync(method, url, body);
      releaseSlot();
      return result;
    } catch (err) {
      if (attempt === MAX_RETRIES) {
        releaseSlot();
        throw err;
      }
      const delay = RETRY_BASE_DELAY_MS * Math.pow(2, attempt);
      console.warn(
        `[tv-token] Request failed (attempt ${attempt + 1}/${MAX_RETRIES + 1}), retrying in ${delay}ms:`,
        method, url.replace(/access_token=[^&]+/, "access_token=***"),
        err instanceof Error ? err.message : err
      );
      await sleep(delay);
    }
  }
  throw new Error("Unreachable");
}
```

**Критические правила (из RESEARCH.md pitfalls):**
- `requestQueue = new Promise(...)` ОБЯЗАТЕЛЬНО до первого `await` — иначе race condition
- `releaseSlot()` вызывается и при успехе, и при последнем throw — иначе deadlock
- Retry остаётся ВНУТРИ захваченного слота mutex — при ошибке лучше задержать очередь

**НЕ менять:** sleep(), константы, auth flow (fetchNewGuestToken, reloginWithCachedCredentials), getStreamUrl, getChannels, getNovinki, fetchCurrentPrograms, все интерфейсы и типы.
