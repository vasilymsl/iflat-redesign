# iFlat TV-шлейфы

## What This Is

Интеграция двух горизонтальных контентных шлейфов (каруселей) на сайт iFlat — провайдера интернета и ТВ. Шлейфы «Бесплатные ТВ-каналы» (с HLS-видеоплеером при наведении) и «Новинки» (постеры фильмов) размещаются на странице /tv и главной странице. Контент и стримы подтягиваются через API 24h.tv. Визуальное поведение должно воспроизводить UX сайта 24h.tv — расширение карточки при hover с перекрытием соседних.

## Core Value

При наведении на карточку канала пользователь мгновенно видит живой эфир — это главная фишка, которая должна работать безупречно.

## Current Milestone: v1.1 QRATOR Stability Fix

**Goal:** HLS-стримы и расписания каналов стабильно работают, не падая из-за QRATOR rate-limiting

**Target features:**
- Задержки между запросами к API (обход rate-limit)
- Retry с exponential backoff при блокировке
- Проактивное обновление токена до истечения
- Circuit breaker для batch-запросов расписаний

### Out of Scope

- Полноценный видеоплеер с управлением (play/pause/fullscreen) — только превью при hover
- Авторизация пользователей 24h.tv — используем гостевой токен
- Кеширование стримов/данных на бэкенде — клиентское кеширование достаточно
- Мобильные жесты (swipe) — нативный scroll достаточен
- Воспроизведение видео на мобильных при hover — hover нет на тачскринах

## Context

- **Проект**: Next.js (App Router) с Tailwind v4, запущен на localhost:3333
- **Существующий код**: базовые компоненты шлейфов уже созданы, но hover-эффект примитивный (scale 1.05 вместо 140% расширения)
- **API 24h.tv**: гостевой токен через `POST /v2/users` с `{ is_guest: true }`, каналы через `GET /v2/channels`, стримы через `GET /v2/channels/{id}/stream`
- **CDN**: изображения с `cdn.media.24h.tv` работают без авторизации
- **Известные проблемы**: SSL/npm в текущей сети (strict-ssl false), Google Fonts блокируются при build
- **4 канала** (Россия 1, Россия 24, ТВЦ, СТС) используют логотип как thumbnail — нет реальных превью

## Constraints

- **API**: HLS-стримы 24h.tv требуют access_token — необходим гостевой токен
- **CORS**: стримы могут блокироваться CORS — нужна проверка и возможный прокси
- **Сеть**: SSL-сертификаты в текущей сети самоподписанные — npm strict-ssl false
- **Совместимость**: hls.js для Chrome/Firefox, нативный HLS для Safari

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| hls.js вместо video.js | Легковеснее, нужен только HLS | — Pending |
| Нативный scroll вместо Embla | Проще, меньше зависимостей, scroll-snap | ✓ Good |
| CDN 24h.tv для изображений | Работает без авторизации, быстро | ✓ Good |
| Гостевой токен 24h.tv для стримов | Единственный способ получить HLS-потоки без полной авторизации | ✓ Good |
| CSS-анимация для hover (не JS) | Плавность, производительность GPU | ✓ Good |
| curl вместо fetch для API 24h.tv | QRATOR блокирует Node.js по TLS fingerprint (JA3) | ✓ Good |
